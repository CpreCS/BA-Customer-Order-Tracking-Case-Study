# Customer Order Tracking Portal — Business Analysis Case Study

## Overview
Business Analysis case study focused on defining requirements, mapping workflows, and designing a self-service solution to address declining customer satisfaction due to lack of order visibility.

---

## Key Contributions
- Broke down the customer order tracking issue and identified its impact on support and retention  
- Gathered and organized requirements based on stakeholder discussions  
- Created epics, user stories, and acceptance criteria to define the solution  
- Mapped the current and future workflow to visualize the improvement  
- Considered system limitations and focused on a simple MVP approach  
- Outlined a solution aimed at reducing support dependency and improving user experience
